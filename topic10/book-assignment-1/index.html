Assignment completion guidelines

  • Submission material:

    • Complete the assignment by modifying the classes in the folder assignments-1.
      • This folder will have been generated when you expanded the file assignment-1.zip that you downloaded.
    • When you are ready to submit, compress the folder assignments-1.
      • Before you compress the folder ensure that the downloaded zip file is not present in your assignments folder otherwise naming confusion is likely to occur.
    • Upload your completed source code in the single file assignment-1.zip.

  • All source code that your write, including individual methods and constructors, is to be documented using as a model the Cone class provided in the project shapes.

    • Marks will be awarded for documentation and presentation.
    • Document only the source code that you write.
      • Do not document legacy code.
    • Avoid naive comments.
      • Example: //here we are allocating memory to the variable house (bad)_
    • Use the minimum number of comments necessary.
    • It is not necessary to comment all fields and methods.
      • Where possible let the choice of variable and method name serve as documentation.
      • No need to document setters or getters.
  • The coding convention you use should match that in the Cone class.
    • Indent 4 spaces.
    • Use self-documenting variable and method names.
      • Not terse
        • Example: String a (bad).
      • But not too long
        • Example: String theBankManagersTemporaryAssistant (definitely too long).
      • How about:
        • Example: String managerAssistant
    • Use blank lines sparingly.
