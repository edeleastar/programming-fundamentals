Guidelines

  • Submission material:

    • Complete the assignment by modifying the classes in the folder assignments-2 as requested in the questions.
      • This folder will have been generated when you expanded the file assignment-2.zip that you downloaded.
    • When you are ready to submit, compress the folder assignments-2.
      • Before you compress the folder ensure that the downloaded zip file is not present in your assignments folder otherwise naming confusion may occur.
    • Upload your completed source code in the single file assignment-2.zip
      • Do not change the name of the file (such as by adding your student number and so on)
      • Each file is automatically located in individual student directories so that files having similar names will not overwrite each other.

  • All source code that your write, including individual methods and constructors, is to be documented using as a model the Cone class provided in the project shapes.

    • Marks will be awarded for documentation and presentation.
    • Document only the source code that you write.
      • Do not document legacy code unless specifically requested.
    • Avoid naive comments.
      • Example: //here we are allocating memory to the variable house (bad)_
    • Use the minimum number of comments necessary.
    • It is not necessary to comment all fields and methods that you write.
      • Where possible let the choice of variable and method name serve as documentation.
      • Unlike in assignment 1, you are asked to comment any getters or setters that you may write.
    • Do not mix inline and block commenting styles
    • Use the JavaDoc commenting style where appropriate:
/**
 *
 */
  • Further information on code styling is in JavaCodeConventions available in the Slides section of this assignment.
  • The coding convention you use should match is that in the Cone class.
    • Indent 4 spaces.
    • Use self-documenting variable and method names.
      • Not as terse as:
        • Example: String a
      • But not too long
        • Example: String theBankManagersTemporaryAssistant (definitely too long).
      • How about:
        • Example: String managerAssistant
      • It is acceptable to use single character variable names for loop counters.
    • Use blank lines sparingly.
